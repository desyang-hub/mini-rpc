## 实现动态代理和远程调用

### 第 5-7 天：动态代理与 RPC 调用流程
任务：
* 服务端：维护一个 ServiceMap（哈希表），Key是"服务名.方法名"，Value是函数指针/回调。收到请求后查表执行。
* 客户端：封装 RpcChannel。用户调用 stub->Method() -> 序列化 -> 发送网络请求 -> 等待响应 -> 反序列化 -> 返回。
* 难点：同步与异步。先实现同步调用（阻塞等待），有余力再上 std::future 或回调实现异步。



### 遇到的问题

对于这里的实现，也就是RpcServer, 这里的handlers_起初声明为静态成员变量，结果出问题了，当使用RPC_SERVICE_BIND绑定时，使用的也是静态类初始化来实现服务方法注册，而这个时候handlers_未必已经初始化好了，有个先后；
解决方案：使用静态成员方法返回静态局部变量获取handlers_
